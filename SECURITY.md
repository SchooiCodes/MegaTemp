# Security Policy

## Supported Versions

Only the latest release on the `master` branch is supported with security
fixes.

| Version | Supported |
| --- | --- |
| latest `master` | ✅ |
| older releases | ❌ |

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not** open a public
issue. Instead, report it privately:

- Use GitHub's private vulnerability reporting on the
  [Security tab](https://github.com/SchooiCodes/MegaTemp/security/advisories/new),
  **or**
- Email the maintainer (see the profile linked from the repository).

You can expect an acknowledgement within a few days. Please give us a
reasonable amount of time to investigate and prepare a fix before any public
disclosure.

## Scope note

MegaTemp automates third-party services (mail.tm, mega.nz). Security issues
that live entirely within those external services are out of scope here — please
report them to the respective providers.
